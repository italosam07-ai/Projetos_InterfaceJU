# Objetivo do projeto
Criar um Prototipo de funcionalidade adcional para uma ferramenta já existende que faça a criação e gerenciamento de eventos
# Descrição do projeto
### Visão Geral
O projeto consiste em um wireframe de baixa fidelidade que introduz um sistema nativo de gestão de eventos dentro do WhatsApp. O objetivo é eliminar a necessidade de aplicativos externos ou "votações" informais em conversas, permitindo que os usuários criem, gerenciem e acompanhem confirmações de presença diretamente em uma nova aba dedicada.
## Detalhamento dos Fluxos de Tela: Extensão de Eventos
O ecossistema foi desenhado para ser circular, permitindo que o usuário transite entre a conversa e a gestão do evento sem fricção.

1. Fluxo de Descoberta e Entrada (Tela Inicial e Hub)
Acesso Global: Através da barra de navegação inferior, o usuário acessa o Hub de Eventos. Esta tela centraliza todos os compromissos, exibindo cards resumidos com o nome do evento e o status rápido de engajamento (Confirmados vs. Pendentes).

Integração de Mensagens: Dentro de uma conversa individual ou de grupo, um novo ícone de "+" (Evento) no cabeçalho serve como atalho contextual para criar um evento diretamente com aqueles participantes.

2. Fluxo de Criação e Configuração
Estrutura de Cadastro: O usuário define a identidade visual (foto de capa), o nome e a descrição detalhada.

Input de Dados: Campos específicos para Data e uma lista prévia de Convidados.

Adição Dinâmica: A tela de "Adição de Pessoas" utiliza o padrão nativo de seleção múltipla com checkbox e busca, garantindo familiaridade e velocidade na montagem da lista de presença.

3. Fluxo de Gerenciamento Detalhado (Tela do Evento)
Esta é a "central de comando" de um evento já criado:

Ações Rápidas: Botões de ação direta para Ligar, Vídeo, Adicionar Pessoas ou acessar as Configurações do evento.

Painel de Informações: Exibição clara do local, link ou descrição, acompanhada de uma linha do tempo ou blocos de notas (vistos nos placeholders centrais).

Controle de Presença: Uma lista detalhada de participantes onde o organizador pode visualizar o status individual (confirmado com ícone de check ou pendente).

Configurações Avançadas: Opções de alternância (toggles) para funções como "Mensagens Automáticas" ou "Privacidade Avançada", permitindo personalizar a experiência de notificação do grupo.

4. Fluxo de Contexto de Grupo
Banner de Evento: No topo de uma conversa de grupo, o evento aparece como um elemento fixado ou um "header" especial, permitindo que os membros acessem os detalhes sem precisar buscar no histórico de mensagens.

Diferenciais de UX Identificados
Hierarquia de Informação: O uso de avatares redondos para participantes e ícones padronizados mantém a consistência com a linguagem visual do WhatsApp.

Feedback Visual: A diferenciação clara entre "Confirmados" e "Pendentes" no dashboard inicial economiza toques, entregando o valor da informação logo no primeiro nível de navegação.
